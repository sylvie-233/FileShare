# 安全政策

## 支持的版本

目前，我们仅支持最新版本的 FileShare。

## 报告漏洞

如果你发现了安全漏洞，请不要公开 issue。请通过以下方式私密地报告：

- 发送邮件至：security@example.com
- 使用 GitHub 的私有漏洞报告功能

请包含以下信息：

- 漏洞描述
- 受影响的版本
- 复现步骤
- 潜在影响
- 如果可能，提供修复建议

## 安全最佳实践

### 对于用户

1. **网络安全**
   - 仅在可信的局域网环境中使用
   - 不要在公共网络中暴露服务
   - 使用防火墙限制访问

2. **文件安全**
   - 不要上传敏感文件
   - 定期清理不需要的文件
   - 注意文件权限设置

3. **访问控制**
   - 当前版本没有用户认证
   - 建议在内网环境中使用
   - 未来版本将添加用户认证功能

### 对于开发者

1. **依赖管理**
   - 定期更新依赖包
   - 检查已知漏洞
   - 使用 `uv audit` 检查依赖

2. **代码安全**
   - 验证所有用户输入
   - 避免代码注入
   - 使用参数化查询

3. **数据保护**
   - 不要记录敏感信息
   - 安全存储用户数据
   - 使用 HTTPS（生产环境）

## 已知安全问题

目前没有已知的安全问题。

## 安全更新

当发现安全漏洞时，我们会：

1. 确认漏洞
2. 评估影响范围
3. 开发修复补丁
4. 发布安全更新
5. 通知用户

## 责任披露

我们承诺：

- 在 48 小时内确认收到漏洞报告
- 在 7 个工作日内提供初步评估
- 在修复后及时发布安全更新
- 适当致谢安全研究者
